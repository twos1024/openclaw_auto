# ClawDesk v0.1.4

## Highlights

- Fixed Tauri v2 runtime detection so packaged desktop builds no longer misidentify themselves as browser preview mode.
- Switched the frontend command bridge to `@tauri-apps/api/core`, keeping `window.__TAURI__` only as a compatibility fallback.
- Split runtime handling into explicit states for browser preview, healthy desktop runtime, and desktop runtime bridge failure.
- Added runtime diagnostics to help distinguish "opened in preview" from "desktop shell is present but invoke bridge is broken".

## Included Changes

- Added `RuntimeDiagnostics` and three-state runtime detection to the frontend API layer.
- Updated Install, Config, Logs, Settings, Service, Dashboard, and Overview flows to stop rewriting native runtime failures as preview mode.
- Added unit and integration coverage for runtime detection, desktop-bridge-unavailable flows, and overview/runtime diagnostics behavior.
- Verified the Windows packaged installer launches into `OpenClaw Runtime Overview` instead of preview-mode fallback.
