# ClawDesk v0.3.1

## Highlights

- Fixed a critical Windows bug where the OpenClaw binary could not be spawned after installation (OS error 193 / 0xc0000142).
- Improved PATH discovery for WinGet-installed and nvm-windows Node.js environments.
- Added missing progress bounds boundary test per spec requirement.

## Bug Fixes

- **Windows binary resolution (OS error 193):** npm creates both a bare Unix shell script (`openclaw`) and a `.cmd` batch wrapper (`openclaw.cmd`). The `where` command and `resolve_binary_path()` could resolve to the non-executable shell script first, causing all subsequent `openclaw` commands to fail with "not a valid Win32 application". Fixed by:
  - Searching for `openclaw.cmd` instead of `openclaw` in `locator_command()`.
  - Removing the bare `openclaw` (no extension) fallback from `binary_candidates()`.
  - Preferring `.cmd`/`.exe` paths when `where` returns multiple results in `path_from_locator_output()`.
  - Reusing the corrected parser in `detect_env()` to eliminate duplicated logic.

## Enhancements

- **Broader Windows PATH discovery:** Added `%LOCALAPPDATA%\npm` and dynamic Node.js installation directory detection to `desktop_runtime_bin_dirs()`, improving compatibility with WinGet and nvm-windows setups.
- **Progress bounds test:** Added explicit boundary test verifying running progress stays within 18–95% range, closing a spec coverage gap from `install-ui-progress.md`.
