# ClawDesk v0.3.2

## Highlights

- Fixed visible console window popups on Windows — all child processes now spawn silently.
- Made `where.exe` / `which` failure non-fatal — the app no longer crashes when the system locator is broken.
- Reversed binary resolution priority: filesystem scan first, `where`/`which` as fallback only.

## Bug Fixes

- **Console window popups (CREATE_NO_WINDOW):** Every `Command::new()` call from a Tauri GUI app was opening a visible terminal window on Windows. Added `CREATE_NO_WINDOW` (0x08000000) creation flag to both `shell::run_command()` (async, tokio) and `platform::which_sync()` (sync, std).

- **`where.exe` crash (0xc0000142) no longer fatal:** On some Windows systems, `where.exe` itself crashes with error 0xc0000142 ("application failed to initialize properly"). Previously this caused `detect_env()` to return an error and blocked the entire app. Now:
  - `detect_env()` treats the locator command as optional — uses filesystem-based `resolve_binary_path()` as primary, `where`/`which` as secondary fallback.
  - `ensure_openclaw_available()` also gracefully handles locator failure.
  - `locator_output` field in `DetectEnvData` is now `Option<ShellOutput>` instead of required.

- **`version_output` error no longer blocks detect_env():** If `openclaw --version` fails after the binary is found, the app now continues with `openclaw_version: null` instead of returning an error.
