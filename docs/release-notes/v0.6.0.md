# ClawDesk v0.6.0

## Highlights

- Stopped misclassifying backend aggregation failures as desktop runtime outages, while still rendering a usable recovery overview with the real error surfaced to the user.
- Preserved unsaved user input when Settings writes fail, so retry flows no longer wipe the draft form state.
- Reloaded Config state after successful saves so stale "using defaults / config missing" warnings clear immediately.
- Tightened the Rust boundary again: live Runbook and Setup Assistant now consume backend-generated runbook payloads directly instead of front-end secondary derivation.

## Stability

- `statusService` now distinguishes backend aggregation failures from `preview` and `runtime-unavailable` runtime modes.
- `runbookService` only builds local fallback models for preview/runtime-offline cases and surfaces live backend failures explicitly.
- `useSettingsForm()` only reloads persisted settings after successful writes; failed saves keep the current draft intact.
- `useConfigForm()` now refreshes its source snapshot after a successful save so page-level warnings and file metadata stay in sync.
- Type boundaries and E2E fixtures were cleaned up so `tsc --noEmit`, unit tests, Rust tests, and Playwright runs all pass against the same model contracts.
