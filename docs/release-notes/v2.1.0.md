# ClawDesk v2.1.0

## Highlights

- Fixed a security gap where Content Security Policy was completely disabled (`null`), leaving the Dashboard iframe with zero XSS protection. CSP now restricts all resources to same-origin and explicitly allows only `localhost:*` connections for Gateway HTTP/WebSocket and iframe access.
- Removed the legacy `instance` command layer (580 lines of dead code) — Rust commands, service module, and the two unused frontend stub files that were never imported by the router.
- CI now catches Windows-specific Rust path-handling bugs: a new `rust_windows` job runs `cargo test --lib` on `windows-latest` in parallel with the existing Ubuntu lint job.

## Security

- `tauri.conf.json`: replaced `"csp": null` with a scoped policy — `default-src 'self'`, explicit `connect-src http://localhost:* ws://localhost:*` for Gateway, and `frame-src http://localhost:*` for the Dashboard iframe.

## Removed

- `src-tauri/src/commands/instance.rs` — legacy IPC command layer, unused since v2.0 agent migration.
- `src-tauri/src/services/instance_service.rs` — underlying service, 512 lines, fully superseded by `agent_service`.
- `src/pages/InstancesPage.tsx` and `TokenPage.tsx` — dead re-export stubs never imported by the router.

## CI

- Added `rust_windows` job: runs `cargo test --lib` on `windows-latest` to surface Windows path-handling issues that Ubuntu CI misses.
- Renamed `lint_and_test` job to `frontend` and updated step labels to reflect that `npm run test:unit` covers both unit and integration tests.

## Verification

- `npm run lint` — frontend ESLint + Rust fmt check pass
- `npm run test:unit` — 18 test files, 83 tests pass (unit + integration)
