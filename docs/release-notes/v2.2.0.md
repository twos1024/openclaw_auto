# ClawDesk v2.2.0

## Highlights

- Added a shell-level top bar with page titles and a global `Setup Assistant` entry so onboarding is reachable from anywhere in the app.
- Added the `Dashboard` navigation item and kept the embedded Dashboard flow aligned with the existing Gateway/runtime state.
- Completed edit flows for Providers, Channels, and Cron jobs so existing records can be changed without recreating them.
- Tightened backend service behavior so CLI failures surface as real errors instead of fake success states.

## Changed

- `AppShell` now renders a route-aware header and exposes `Setup Assistant` as a first-class action.
- `Sidebar` now includes `Dashboard` in the primary navigation.
- `SetupPage` refreshes Gateway status when the flow enters the Gateway / completion steps.
- Provider, Channel, and Cron dialogs now support edit mode in addition to create mode.

## Fixed

- CLI command failures now propagate through the service layer.
- `apiKeyRef`, `baseUrl`, and related configuration values are preserved on create/update flows.
- Instance IDs now use UUID v4 generation instead of timestamp-based fallback IDs.
- Setup Assistant and Dashboard states no longer rely on stale status when Gateway is already running.

## Verification

- `npm run lint:frontend`
- `npm run test:unit`
