# OpenClaw Manager v2.0.0 — 产品需求文档 (PRD)

> 基于 ClawX 项目架构全面重构，对齐 Agent/Channel/Provider 模型。
> 日期：2026-03-19
> 版本：v2.0.0

---

## 1. 产品定位

| 维度 | 描述 |
|------|------|
| 产品名称 | OpenClaw Manager |
| 一句话定义 | 面向开发者与团队的本地多智能体、多渠道 AI 网关管理桌面应用 |
| 核心价值 | 在本地统一管理多个 AI Agent，对接多模型供应商、多消息渠道，提供对话调试、定时任务与用量监控 |
| 品牌归属 | OpenClaw（开源 AI 网关） |
| 参考项目 | ClawX (https://github.com/ValueCell-ai/ClawX) |
| 当前版本 | v1.0.0 |
| 目标版本 | v2.0.0（本文档所描述的重构版） |

### 1.1 产品愿景

OpenClaw Manager v2.0 的目标是成为开发者和技术团队管理本地 AI 智能体的首选桌面工具。通过对齐 ClawX 的 Agent/Channel/Provider 三层架构，用户可以在一个统一界面中完成以下核心工作流：

1. **创建和配置 AI Agent** — 定义系统提示词、选择底层模型、分配消息渠道
2. **连接多种消息渠道** — 将 Agent 接入 WhatsApp、Telegram、Discord、钉钉、飞书、企业微信等 14 种渠道
3. **管理模型供应商** — 集中配置 OpenAI、Anthropic、DeepSeek、Ollama 等供应商的 API 密钥
4. **调试和监控** — 内置对话界面、用量统计、定时任务管理

## 2. 目标用户画像

| 角色 | 场景 | 痛点 | v2.0 解决方案 |
|------|------|------|---------------|
| 独立开发者 | 在本地搭建私有 AI 助手，快速切换模型调试 | 不想写代码管理多个 API Key 和 Prompt；缺乏可视化工具 | Agent 向导 + Provider 集中管理 + 内置对话调试 |

| 运维工程师 | 监控 AI 服务运行状态，追踪 Token 消耗，执行定期维护任务 | 缺乏用量可视化和自动化定时任务能力 | Token 用量统计仪表盘 + Cron 定时任务模块 |

## 3. 核心概念模型

v2.0 采用 ClawX 的三层架构模型，明确各实体之间的关系：

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenClaw Manager v2.0                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────┐      ┌──────────┐      ┌──────────┐          │
│   │ Provider │──1:N──│  Agent   │──N:N──│ Channel  │         │
│   │ (供应商)  │      │ (智能体)  │      │  (渠道)   │         │
│   └─────────┘      └──────────┘      └──────────┘          │
│        │                │                  │                │
│   API Key 管理     系统提示词/模型      WhatsApp/Telegram    │
│   模型列表获取     技能(Skill)绑定     Discord/钉钉/飞书     │
│   连通性验证       Cron 定时任务       企业微信/...          │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                    OpenClaw Gateway                          │
│              (独立进程 — 网关运行时)                           │
└─────────────────────────────────────────────────────────────┘
```

| 概念 | 定义 | 来源 |
|------|------|------|
| **Agent (智能体)** | 一个独立的 AI 对话实体，包含系统提示词、模型配置、技能绑定和渠道分配 | ClawX Agents |
| **Channel (渠道)** | 消息通道，将 Agent 连接到外部平台（如 WhatsApp、Telegram、钉钉等） | ClawX Channels |
| **Provider (供应商)** | AI 模型服务供应商，管理 API 密钥和可用模型列表 | ClawX Providers |
| **Skill (技能)** | Agent 可调用的工具或函数，通过市场安装或自定义创建 | ClawX Skills |
| **Cron (定时任务)** | 按 cron 表达式定期执行的自动化任务 | ClawX Cron |
| **Gateway (网关)** | OpenClaw 运行时进程，负责 Agent 编排、消息路由和技能执行 | OpenClaw Core |

## 5. 功能模块详细需求

### 5.1 智能体管理 (Agent Management)

**概述：** 核心模块。用户在此创建、编辑、删除和管理 AI 智能体。每个 Agent 是一个独立的对话实体，绑定特定的模型、系统提示词、技能集合和消息渠道。

#### 5.1.1 智能体列表页

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| Agent 卡片列表 | 以卡片形式展示所有智能体，含名称、头像、状态指示、绑定模型、已连接渠道图标 | P0 |
| 搜索过滤 | 顶部搜索栏，按名称模糊搜索；支持按状态（运行中/已停止）筛选 | P0 |
| 新建入口 | 右上角「+ 新建智能体」按钮，进入创建向导 | P0 |
| 空状态引导 | 无智能体时显示引导插图和「创建你的第一个 Agent」按钮 | P1 |
| 快捷操作 | 卡片上的启动/停止/编辑/删除按钮 | P0 |
| 批量操作 | 多选后批量启动/停止/删除 | P2 |
| 运行概览 | 侧边栏底部展示 Agent 总数和运行中数量 | P1 |

#### 5.1.2 新建智能体向导

| 步骤 | 名称 | 内容 | 说明 |
|------|------|------|------|
| Step 1 | 基本信息 | 名称、头像选择、描述 | 支持中英文名称 |
| Step 2 | 系统提示词 | System Prompt 编辑器 | 支持 Markdown 编辑；提供预设模板库（翻译专家、代码助手、客服、文案写手等） |
| Step 3 | 模型选择 | 选择 Provider 和具体模型 | 展示模型参数预览（上下文长度、能力标签）；支持参数调整（temperature、max_tokens） |
| Step 4 | 渠道分配 | 选择该 Agent 要接入的渠道 | 多选已配置的 Channel；展示渠道连接状态 |
| Step 5 | 技能配置 | 选择该 Agent 可用的技能 | 从已安装技能中勾选 |

#### 5.1.3 智能体详情/编辑

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 配置编辑 | 修改名称、提示词、模型、渠道、技能 | P0 |
| 状态监控 | 运行状态、启动时间、处理消息数 | P0 |
| 对话入口 | 快速跳转该 Agent 的对话界面 | P1 |
| Agent 克隆 | 基于现有 Agent 快速创建副本 | P2 |
| 配置导出/导入 | JSON 格式导出和导入 Agent 配置 | P2 |
| 多 Agent 路由 | 支持 @agent 语法在对话中路由到其他 Agent | P2 |

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | Agent 列表 500 条以内首屏加载 <300ms；对话流式输出延迟 <100ms |
| 内存 | 应用空闲状态内存占用 <150MB；网关进程独立管控，无泄露 |
| 安装包 | Windows x64 安装包 <80MB |
| 离线 | 核心管理功能离线可用；对话和市场功能需网络；Ollama 本地模型支持完全离线对话 |
| 安全 | API Key 使用操作系统原生密钥链加密存储；日志中不输出敏感信息；渠道 Token 加密存储 |
| 平台 | Windows (x64) 为主，macOS (x64/arm64) 和 Linux (x64) 为辅 |
| 进程管理 | 网关进程独立监控；应用退出时确保所有子进程正确清理；异常崩溃自动重启 |
| 无障碍 | 关键交互元素支持键盘导航和屏幕阅读器 |

## 10. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 渠道 API 变更频繁 | 部分渠道接入可能因第三方 API 变更而失效 | 抽象渠道适配器层；优先支持稳定 API 的渠道 |
| 供应商 API 兼容性 | 不同供应商 API 格式差异较大 | 采用 OpenAI 兼容格式作为统一标准；非兼容供应商单独适配 |
| 网关进程稳定性 | 网关崩溃导致所有 Agent 不可用 | 独立进程监控 + 自动重启 + 健康检查机制 |
| 安装包体积增长 | 支持更多功能后安装包可能超标 | 核心功能内置，技能和插件按需下载 |
| i18n 翻译质量 | 三语翻译维护成本高 | 优先保证中英文质量；日文由社区贡献 |
