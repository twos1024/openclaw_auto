# OpenClaw Manager v2.0 — 产品需求文档 (PRD)

> 基于 v1.0.70 UI 截图逆向分析，输出重构版产品定义。
> 日期：2026-03-19

---

## 1. 产品定位

| 维度 | 描述 |
|------|------|
| 产品名称 | OpenClaw Manager |
| 一句话定义 | 面向开发者与团队的本地 AI 机器人网关管理桌面应用 |
| 核心价值 | 在本地统一管理多个 AI 实例（Bot），对接多模型、多渠道，提供对话调试与用量监控 |
| 品牌归属 | OpenClaw, powered by APIMart |
| 当前版本 | v1.0.70 (openclaw v2026.3.13) |
| 目标版本 | v2.0.0（本文档所描述的重构版） |

---

## 2. 目标用户画像

| 角色 | 场景 | 痛点 |
|------|------|------|
| 独立开发者 | 在本地搭建私有 AI 助手，快速切换模型调试 | 不想写代码管理多个 API Key 和 Prompt |
| 技术团队 Lead | 为团队部署统一的 AI 网关，监控 Token 消耗 | 需要可视化管理面板而非纯 CLI |
| AI 应用产品经理 | 创建不同人设的 Bot 实例，测试对话效果 | 需要无代码创建 Bot 并即时验证 |

## 4. 功能模块详细需求

### 4.1 实例管理

**概述：** 核心模块。用户在此创建、管理、启停 AI Bot 实例。

#### 4.1.1 实例列表页

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 实例卡片列表 | 以卡片形式展示所有实例，含名称、状态、模型、最后活跃时间 | P0 |
| 搜索过滤 | 顶部搜索栏，按名称模糊搜索 | P0 |
| 新建入口 | 右上角「+ 新建实例」按钮 | P0 |
| 空状态引导 | 无实例时显示引导图标和文案 | P1 |
| 批量操作 | 批量启动/停止/删除 | P2 |
| 今日概览 | 侧边栏底部展示实例总数和运行中数量 | P1 |

#### 4.1.2 新建实例向导

截图显示为 4 步向导：

| 步骤 | 名称 | 内容 | 说明 |
|------|------|------|------|
| Step 1 | 名称 | 显示名称输入框 | 支持中文，留空则使用实例 ID |
| Step 2 | 身份 | System Prompt / 人设配置 | 定义 Bot 的角色和行为 |
| Step 3 | 模型 | 选择底层 LLM 模型 | 如 GPT-5.3 Codex 等 |
| Step 4 | 渠道 | 配置 API Key / 接入渠道 | 对接 APIMart 或自有 Key |

**重构建议：**
- 增加 Step 0「模板选择」— 提供预设模板（翻译专家、代码助手、客服等）
- Step 2 增加 Prompt 模板库
- Step 3 增加模型参数预览（上下文长度、价格、能力标签）
- Step 4 增加连通性测试

#### 4.1.3 实例详情/编辑

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 实例配置编辑 | 修改名称、Prompt、模型、渠道 | P0 |
| 实例启停控制 | 启动/停止/重启 | P0 |
| 实例状态监控 | 运行状态、PID、启动时间、端口 | P0 |
| 对话历史入口 | 快速跳转该实例的对话记录 | P1 |
| 实例克隆 | 基于现有实例快速创建副本 | P2 |
| 实例导出/导入 | JSON 格式导出配置 | P2 |

---

### 4.2 AI 对话

**概述：** 内置对话调试界面，支持 LLM 对话和 Agent 模式。

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 模型选择器 | 顶部下拉选择当前对话模型（如 GPT-5.3 Codex） | P0 |
| LLM 对话模式 | 标准多轮对话 | P0 |
| Agent 模式 | 带工具调用的 Agent 对话 | P0 |
| 新对话 | 左侧「新对话」按钮创建新会话 | P0 |
| 对话记录 | 左侧历史会话列表 | P0 |
| 对话参数配置 | 右上角设置图标，配置 temperature、max_tokens 等 | P1 |
| 语音输入 | 底部麦克风按钮 | P2 |
| 消息附件 | 支持图片/文件上传 | P1 |
| Markdown 渲染 | 对话内容支持 Markdown、代码高亮 | P0 |
| 流式输出 | SSE 流式显示生成内容 | P0 |

**重构建议：**
- 对话页应支持绑定到特定实例（使用该实例的模型和 Prompt）
- Agent 模式应展示工具调用过程的可视化
- 增加对话导出功能（Markdown/JSON）

---

### 4.3 Token 概览

**概述：** 查看 API 用量和 Token 消耗统计。

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| API Key 输入 | 输入 APIMart API Key 查询 | P0 |
| 余额查询 | 显示 Key 的剩余余额 | P0 |
| 使用统计 | 近 30 天 Token 使用统计图表 | P0 |
| 按模型统计 | 分模型查看用量 | P1 |
| 按实例统计 | 分实例查看用量 | P1 |
| 用量告警 | 余额低于阈值时提醒 | P2 |
| 费用预估 | 根据历史用量预估月度开销 | P2 |

**重构建议：**
- API Key 应集中在「设置」中管理，Token 页直接读取
- 增加用量趋势图（折线图）
- 增加按日/周/月维度切换

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 实例列表 1000 条以内首屏 <500ms |
| 内存 | 空闲状态内存占用 <200MB（修复当前 Node.js 泄露） |
| 安装包 | Windows x64 安装包 <100MB |
| 离线 | 核心管理功能离线可用，对话功能需网络 |
| 安全 | API Key 本地加密存储，不明文写日志 |
| 平台 | Windows (x64) 为主，macOS / Linux 为辅 |

---

## 6. v2.0 重构重点

| 编号 | 重构项 | 原因 |
|------|--------|------|
| R1 | 修复 Node.js 进程泄露 | 当前版本 useGatewayControl 竞态条件导致 interval 泄露 |
| R2 | 实例向导增加模板 | 降低新用户上手门槛 |
| R3 | Token 统计可视化 | 当前仅文字展示，缺乏图表 |
| R4 | 对话绑定实例 | 对话页应可选择实例上下文 |
| R5 | API Key 集中管理 | 避免 Token 页和实例创建时重复输入 |
| R6 | 进程生命周期管控 | 应用退出时确保所有子进程清理 |

## 7. 里程碑

| 阶段 | 内容 | 交付物 |
